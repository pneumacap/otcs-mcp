Vector Database: pgvector

Zero new infrastructure — Altius already runs PostgreSQL

Why pgvector Wins for Altius

  • Already running PostgreSQL — single CREATE EXTENSION
  • Zero new vendor, zero new cost, zero new ops
  • Handles 500K+ vectors per table efficiently
  • HNSW index for fast approximate nearest-neighbor
  • Drizzle ORM already supports it
  • Multi-tenant isolation via org_id filtering
  • Self-hostable for on-premise deployments

Implementation Roadmap

6 phases, 10 weeks, 80% of value in the first 4 weeks

Key Takeaways

1. ANTHROPIC STAYS PRIMARY
Claude's tool-use is best-in-class. Multi-provider is about cost optimization and risk mitigation, not replacement.
2. $618K/YEAR SAVINGS AT YEAR 2
Gross margin improves from 83% to 94%. API spend drops 63% through intelligent routing and RAG caching.
3. AGENTS ARE THE BIGGEST LEVER
Moving agent processing from Sonnet ($0.035/doc) to GPT-4.1 nano ($0.005/doc) is an 86% cost reduction.
4. RAG IS FREE INFRASTRUCTURE
pgvector runs inside existing PostgreSQL. Embedding costs are negligible ($80 for 2M docs). Saves $138K/year.
5. ENTERPRISE DEALS BECOME VIABLE
5,000-user deal: $29,842/mo cost → $8,150/mo. Profitable at $10K/mo instead of requiring $40,000/mo.
6. BYOK ELIMINATES API COST
Enterprise customers bring their own API key. License revenue is near-pure margin. Opens gov/defense markets.
7. START WITH ABSTRACTION LAYER
1-2 week investment enables all subsequent optimizations. The tool layer is already portable — only the orchestrator needs work.
8. VENDOR RISK DROPS TO MANAGEABLE
With 3+ providers, no single vendor's pricing or availability can threaten the business. Auto-failover in seconds.
